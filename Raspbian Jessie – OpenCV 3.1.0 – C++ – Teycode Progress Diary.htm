Raspbian Jessie – OpenCV 3.1.0 – C++

Raspbian Jessie – OpenCV 3.1.0 – C++

by Teycode
in Uncategorized
Comments

This will be a short guide, I’ll just update certain parts of my previous OpenCV tutorial. I’m going to assume that; we have a fresh install of Raspbian Jessie on our Raspberry Pi and we are connecting to the internet via ethernet interface. So, lets begin.

 

Preliminary

 

1. Jessie will boot directly to the desktop. A little surprise here, the “raspi-config” is now gone. Instead of that now we have a gui for OS configuration. Go to Menu -> Preferences -> Raspberry Pi Configuration and select Expand Filesystem as a fresh install won’t be using the whole SD card.

 

2. From the Interfaces tab enable the camera since it will be disabled as default.

 

3. Performance tab can stay as it is but sometimes web browser and other graphical related stuff might crash since 64MB for a GPU is extremely small. If that happens you can resize it to 128. I’m leaving it to you. (Although I’ve realized that Raspbian has increased my GPU Memory split over time by itself. Really weird.)

 

4. At Localisation tab you can set a different keyboard layout, just be careful not  to touch Locale and Timezone for now as it has crashed my previous install for unknown reasons. After you’ve completed these steps, reboot.

 

5. When the system boots, do a quick update with:

1
sudo rpi-update

 

6. Now reboot again. One last step remains at this point, just update 3rd party libraries and sources with:

1
sudo apt-get upgrade && sudo apt-get dist-upgrade && sudo apt-get autoremove

The dist-upgrade resolves dependency conflicts and autoremove gets rid of redundant dependencies.

 

Now our install is up to date and ready to play.

 

Installing the UV4L Driver

 

Beginning the install of UV4L driver for Raspicam. This part won’t take too long. Just copy/paste given commands or edit necessary files.

  1. Copy/Paste and run command given below, this command adds UV4L repository key.
    1
    curl http://www.linux-projects.org/listing/uv4l_repo/lrkey.asc | sudo apt-key add -
  2. Run the given command below.
    1
    sudo nano /etc/apt/sources.list

    And copy/paste following line. After that, press “Ctrl + O” to save and “Ctrl + X” to quit. As you might have noticed, we didn’t add the repo as Jessie but as Wheezy because linux-projects does not have a Jessie version of the repo yet.
    1
    deb http://www.linux-projects.org/listing/uv4l_repo/raspbian/ wheezy main

    This section adds UV4L repository so we can install UV4L driver with simple apt-get commands.
  3. Run the two lines given below, now this section might take a few minutes. Doing classic apt-get update and installing UV4L driver.
    1
    2
    sudo apt-get update
    sudo apt-get install uv4l uv4l-raspicam
  4. (Optional) The next line is for making driver to run at boot with a specific configuration set. If you don’t want to start driver manually each time you boot Raspberry Pi, I highly suggest this part. With this package we can restart driver at any time, too.
    1
    sudo apt-get install uv4l-raspicam-extras
  5. Now we will do a system update. Run the given command.
    1
    sudo rpi-update
  6. (Optional) Well, this part is actually not optional if you don’t want a huge preview screen popping up everytime you run your OpenCV program. This part only works if you’ve done part 4.
    First, lets open UV4L configuration file to make the change.
    1
    sudo nano /etc/uv4l/uv4l-raspicam.conf

    And change this line:
    1
    # nopreview = yes

    To this:
    1
    nopreview = yes
  7. (Optional) Finally, if you are considering doing streams or accessing your camera by network you should run commands given here to install extra packages.
    1
    2
    3
    4
    5
    6
    7
    sudo apt-get install uv4l-server
    sudo apt-get install uv4l-uvc
    sudo apt-get install uv4l-xscreen
    sudo apt-get install uv4l-mjpegstream
     
    (Raspberry Pi 2 Only)
    sudo apt-get install uv4l-webrtc
  8. And thats it, lets make a final check here. If you are able to capture a frame with line below, you can proceed to next part.
    1
    dd if=/dev/video0 of=snapshot.jpeg bs=11M count=1

 

Installing OpenCV 3

 

 

The last and the longest part. Nothing much to do here but since we are compiling OpenCV from source, it will take a LONG time. From 2-3 hours (Pi 2) to 3-12 hours. Lets begin!

  1. Copy and execute given command exactly as it is. This is the first block of dependencies and these ones are about compilation process.
    1
    sudo apt-get -y install build-essential cmake cmake-curses-gui pkg-config
  2. Moving on to the second block of dependencies. These ones are the necessary dependencies to compile OpenCV from the source.
    1
    sudo apt-get -y install libjpeg-dev libtiff5-dev libjasper-dev libpng12-dev libavcodec-dev libavformat-dev libswscale-dev libv4l-dev libeigen3-dev libxvidcore-dev libx264-dev libgtk2.0-dev
  3. Lets download OpenCV source code.
    1
    wget http://sourceforge.net/projects/opencvlibrary/files/opencv-unix/3.1.0/opencv-3.1.0.zip/download -O opencv-3.1.0.zip
  4. Execute commands below one by one to set environment.
    1
    2
    3
    4
    5
    unzip opencv-3.1.0.zip
    cd opencv-3.1.0
    mkdir release
    cd release
    ccmake ../

     
  5. Now, we are at the most important part. I will share an example build configuration but please read each one carefully otherwise you might have problems later. I’ve enabled NEON support here because NEON registers can be used in increasing OpenCV performance but I’ve yet to try it. Also if you want to have text fields, buttons and fancy stuff like that within your OpenCV program, install QT before doing this part and enable QT support in build configuration. Also, you can disable examples, documentation etc. to gain time while compiling. It’s your choice.
    Press ‘c’ key to configurate, move around with arrow keys and to change options press enter. When you finish, press ‘c’ again to save configuration and press ‘g’ to generate Makefile. (Do not try to copy/paste this part, read each line and check if it is the same.)

  6. Now the boring part comes, first command takes a lot of time as I’ve said. Second line doesn’t take long. If you have any kind of difficulty while compiling with first line (such as electricity blackouts, OS freeze or crash etc.); just open up the terminal again, go to the opencv-3.1.0/release/  and run make command again. It’ll continue from where it left off. Also, be careful with the -j flag as it could overheat your Pi since it makes g++ to use multiple cores.
    1
    2
    make -j4
    sudo make install
  7. After you’ve executed both commands above, you might get an error like “Error while loading shared libraries. Cannot open shared object file. No such file or directory” while running an OpenCV C++ program. If you get this error, just run the line below.
    1
    sudo ldconfig -v

     

If you didn’t have any problem until this point, congratulations! Now, you have a SD Card with Raspbian + UV4L Cam Driver + OpenCV configured in it. I strongly recommend backing up your SD Card as a precaution at this point. (Be careful, since we’ve expanded file partition, image size of SD Card will be as large as SD Card capacity!)

 

Have fun!
